# Автоматизация тестов: Saby & Tensor (UI Scenarios)

Данный проект содержит автоматизированные UI-тесты для порталов Saby.ru и Tensor.ru. 
Тесты разработаны в рамках выполнения тестового задания на позицию разработчика в тестировании (Программист-тестировщик).

Стек:

- Язык: Python 3.14
- Фреймворк: Pytest
- Инструмент автоматизации: Selenuim WebDriver
- Паттерн: Page Object Model
- Менеджер драйверов: webdriver-manager (автоматическая установка Chrome-драйвера)

## Сценарии тестирования

### Сценарий 1:
- Переход с saby.ru в раздел Контакты.
- Переход на tensor.ru через баннер.
- Проверка наличия блока "Сила в людях".
- Переход в "Подробнее" и валидация размеров изображений в блоке "Работаем" (проверка идентичности height и width).

### Сценарий 2:
- Переход с saby.ru в раздел Контакты.
- Проверка текущего региона и партнеров.
- Проверка обновления списка партнеров, URL и Title страницы на соответствие выбранному региону.

### Сценарий 3:
- Переход в раздел "Скачать локальные версии".
- Скачивание файла в директорию проекта из блока СБИС Плагин.
- Сравнение фактического размера файла с размером, указанным на сайте.

## Запуск проекта

### 1. Подготовка окружения

```bash
# Клонировать репозиторий
git clone https://github.com/ligamsi/tenzor_test
cd tenzor_test

# Создать и активировать виртуальное окружение (рекомендуется)
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
# или
venv\Scripts\activate     # Для Windows

# Установить зависимости
pip install -r requirements.txt
```

### 2. Запуск тестов

```bash
# Запустить все тесты
pytest

# Запустить тесты с подробным выводом (verbose) и логами в консоль
pytest -v -s

# Запустить конкретный сценарий
pytest tests/test_saby_region.py
```

## Особенности реализации

- Динамические ожидания: Использование WebDriverWait для стабильности тестов.
- Парсинг данных: Реализована логика извлечения веса файла из текста ссылок с помощью регулярных выражений.
- Cleanup: В тестах на загрузку реализовано удаление скачанных файлов после выполнения проверок.
- Сложные локаторы: Применение XPath осей (ancestor, following) для работы с динамической версткой.
